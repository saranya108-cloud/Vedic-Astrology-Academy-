# 🤖 FABEL_CONTEXT — Read This First

Hello, AI assistant. This file tells you how to work in this repo safely. Read it before touching anything.

## What this repo is

A personal **AI projects command-center packet**: Markdown notes that track projects, next steps, and progress. It is maintained mostly from an iPhone, synced between Obsidian and GitHub. It also hosts one web app (`index.html`, the Vedic Astrology Academy — see [VEDIC_ASTROLOGY_ACADEMY.md](VEDIC_ASTROLOGY_ACADEMY.md)).

It is **not** a full Obsidian vault, and it must never contain private personal, medical, financial, password, or account information.

## How to read the dashboard

1. Start at [AI_PROJECTS_DASHBOARD.md](AI_PROJECTS_DASHBOARD.md) — project status lives there.
2. Individual projects are notes created from [AI_PROJECT_TEMPLATE.md](AI_PROJECT_TEMPLATE.md), with YAML frontmatter (`type`, `status`, `priority`, `progress`).
3. [PROJECT_LOG.md](PROJECT_LOG.md) is the dated history — newest entries at the top.
4. A project note's **Next Steps** section is the source of truth for what to do next.

## Rules for working here

- ✋ **Ask before deleting, renaming, or reorganizing anything.** No exceptions.
- ✂️ **Prefer small, reviewable edits** — they get reviewed on a phone screen.
- 📝 **Stay in Markdown.** No new tooling, build steps, or plugins unless asked.
- 🔒 **Never add private data**, even if it appears elsewhere in conversation.
- 🧩 **Don't restructure the dashboard format** — consistency matters more than cleverness.
- 📓 When you make a meaningful change, **add a dated line to PROJECT_LOG.md**.

## How to propose changes

1. Say what you want to change and why, in one or two sentences.
2. Make the smallest version of the change that works.
3. If a change touches more than a couple of files, pause and confirm first.
4. Commit messages should be short and descriptive.

When in doubt: ask, keep it small, keep it Markdown.
